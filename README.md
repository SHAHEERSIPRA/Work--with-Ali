# Git Branching, Merging, and Conflict Practice

This repository is created for practicing Git collaboration.

## Purpose
The goal of this project is to practice important Git concepts while working as a team, including:
- Creating and switching branches
- Making commits on different branches
- Merging branches into the main branch
- Understanding and resolving merge conflicts
- Using pull, push, and collaboration workflows

## Team Members
- Muhammad Shaheer
- Muhammad Ali

## Practice Activities
In this repository we will:
1. Create multiple branches.
2. Make different changes in the same files.
3. Merge branches into `main`.
4. Intentionally create merge conflicts.
5. Resolve conflicts and commit the fixes.

## Objective
By the end of this Shaheer will be able to practice, we aim to better understand how Git handles collaboration and how developers work together using branches and merges.
